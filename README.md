# prueba_conflictos
La idea es que cada uno del equipo de alentapp tenga una idea basica de como solucionar conflictos, dado que vamos a estar trabajando en forma paralela y llegado el caso pueden darse conflictos si no se tiene la rama actualizada.


Pasos

crear 2 ramas

se parar en una de las ramas creadas

modificar conflictos.md realizar el push y aunto aceptarse el pr. 

luego cambian a la otra rama creada, y modificarn el mismo archivo, y realizan el push y el pr

en este punto como la segunda rama no esta actualizada con los cambios del main, va a generar conflictos.
